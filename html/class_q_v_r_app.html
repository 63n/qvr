QVR application interface.

=======

The QVR application interface.

>>>>>>> 25ea10d16ed69043417b330980980417c5926455

This is the central interface that VR applications must implement. Most functions provide a default implementation. The only function that must be implemented is render().

Overview:

  • For rendering, implement render().
  • To animate your scene, implement update().
  • To override the default near and far plane, implement getNearFar().
  • To singal when your application wants to quit, implement wantExit().
  • For special process-specific actions, implement initProcess(), exitProcess(), preRenderProcess(), or postRenderProcess().
  • For special window-specific actions, implement initWindow(), exitWindow(), preRenderWindow(), or postRenderWindow().
  • To support your own tracking system that is unsupported by QVR, implement updateObservers().
  • To support multi-process configurations (for multi-GPU or networked rendering), implement serializeDynamicData() and deserializeDynamicData(), and in special cases also serializeStaticData() and deserializeStaticData().
  • To handle keyboard and mouse events, implement keyPressEvent(), keyReleaseEvent(), mouseMoveEvent(), mousePressEvent(), mouseReleaseEvent(), mouseDoubleClickEvent(), or wheelEvent() like you would in any Qt application.

A key concept of QVR is that an application uses only one OpenGL context. This context is available to all functions in the interface, with the exception of the serialization functions.

virtual void QVRApp::render ( QVRWindow *  w,
const QVRRenderContext &  context,
int  viewPass,
unsigned int  texture 
)
pure virtual

Render the current frame.

Parameters
wThe window
contextThe render context with information about the current view
viewPassThe current viewPass for this render call
textureThe texture to render into

This function is called for each window potentially multiple times, once for each view pass, to render the current frame.

The context holds all relevant information that the application needs to render the view. Typically this is the frustum and the view matrix:

QMatrix4x4 P = context.frustum(viewPass).toMatrix4x4();
QMatrix4x4 V = context.viewMatrix(viewPass);

The view must be rendered into the given texture, so the first thing this function typically does is to attach the texture to a framebuffer object that is then used for rendering.
